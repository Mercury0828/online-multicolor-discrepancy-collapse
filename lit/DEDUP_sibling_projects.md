## Overlap analysis
- **Different problems.** Discrepancy of random sparse vectors vs dynamic geometric independent set vs MST-weight
  query complexity. No shared problem, no shared theorem, no shared lemma.
- **No shared object or quantity.** `ondisc_k` (a discrepancy) has nothing to do with MWIS update time or EMST query
  count.
- **No contradiction.** Disjoint domains; none of the three asserts anything that touches another's claims.
- **No shared core technique.** Lovett–Meka / negative association / balls-into-bins (ours) are not used by the other
  two; local-ratio (MIS) and range-counting/WSPD (EMST) are not used by us.
- **Already mutually deduped:** `emst-oracle-lower-bound/PROJECT_STATE.md` itself states it is "NOT overlapping with
  the sibling `dynamic-weighted-mis-fat-objects` project." Our project is a third, equally-disjoint topic.
- **The only commonality is PROCESS, not contribution:** all three were generated by the same research-os
  idea-deepdive workflow, use the same external-solver attack/audit loop, and target SODA 2027 (deadline 2026-07-09).
  Three different papers on three different problems from a common pipeline is **normal** (not self-overlap, not
  salami-slicing) — each stands entirely alone.
